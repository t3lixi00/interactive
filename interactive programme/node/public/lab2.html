1.RWD

1.What is the purpose of the RWD?

To fit in different size of screen. Because there is no design, or fixed or fluid, scales seamlessly beyond the context for which it was originally intended. We need to design for an optimal(最佳的) viewing experience, but embed standards-based technologies into our designs to make them not only more flexible, but more adaptive to the media that renders them.

2.How things are done before RWD? What is the difference?

Before RWD, people usually use desktop to view websites, so it is not a big deal between desktop landscape. Otherwise, flexible design (fluid grid, with flexible images) works well enough in this situation.

3.How css media queries can help you building responsive web sites?

media queries became a popular client-side technique for delivering a tailored style sheet to the iPhone, Android phones, and their ilk.According to different media type, we can set for different and optimal CSS. Thus, when users choosing for various size of screen, the website still appears to optimal version.

Why to use em (font) based sizing for example margins?

Because em is a new function from html5. If you set it for margins, when users using different size of screen, margin will change automatically according to screen size. Because margin would be related to body font size. So when body font size change, the margin will change too.

3.Simple Example1

Question: After you have done the basic layout, how could you prevent user zooming the site on mobile?

The initial-scale property controls the zoom level when the page is first loaded. The maximum-scale, minimum-scale, and user-scalable properties control how users are allowed to zoom the page in or out. For example, initial-scale =1, which means that when users open the website, the scale is normal. if the maximum-scale is 1, which means users cannot scale.
